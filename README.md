    <article class="markdown-body entry-content" itemprop="text"><h1><a id="user-content-muonpogtreeproducer" class="anchor" href="README.md#muonpogtreeproducer" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>MuonPOGtreeProducer</h1>

<p>Muon POG tree producer</p>

<h2><a id="user-content-installation-instructions" class="anchor" href="README.md#installation-instructions" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Installation instructions</h2>

<div class="highlight highlight-source-shell"><pre>cmsrel CMSSW_8_0_20 <span class="pl-c"># Just an example release, works in CMSSW &gt;= 74X at present </span>
<span class="pl-c1">cd</span> CMSSW_8_0_20/src/

git clone https://github.com/calderona/MuonPOG/  (to be updated later) 

cmsenv

scramv1 b -j 5</pre></div>

<h2><a id="user-content-ntuples" class="anchor" href="README.md#ntuples" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Ntuples</h2>

<p>The interface of muon Ntuples is defined in : MuonPOG/Tools/src/MuonPogTree.h</p>

<p>The code filling ntuples is available in : MuonPOG/Tools/plugins/MuonPogTreeProducer.cc</p>

<p>It fills HLT, GEN level, beam spot, vertex and muon information. It works both in AOD and miniAOD (NOTE: trigger information not filled when running in miniAOD).</p>

<p>To create some ntuples :</p>

<div class="highlight highlight-source-shell"><pre><span class="pl-c1">cd</span> MuonPOG/Tools/test/
python muonPogNtuples_cfg.py --print <span class="pl-c"># this will give you the default input parameters of the filler. </span>
                                     <span class="pl-c"># As the ntuple cfg is based on VarParsing you can customise the</span>
                                     <span class="pl-c"># ntuple production via command line [1] or in a crab cfg [2] </span>

[1] 
cmsRun muonPogNtuples_cfg.py globalTag=80X_mcRun2_asymptotic_v5 <span class="pl-cce">\\</span>
  eosInputFolder=/store/relval/CMSSW_8_0_0_patch2/RelValZMM_13/GEN-SIM-RECO/PU25ns_80X_mcRun2_asymptotic_v5_refGT-v1/10000

[2] 
https://twiki.cern.ch/twiki/bin/view/CMSPublic/CRAB3ConfigurationFile#CRAB_configuration_parameters (find pyCfgParams)</pre></div>

<h2><a id="user-content-running-on-crab" class="anchor" href="README.md#running-on-crab" aria-hidden="true"><svg aria-hidden="true" class="octicon octicon-link" height="16" version="1.1" viewBox="0 0 16 16" width="16"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Running on CRAB</h2>

<p>For running trhough crab you can go to: </p>

<pre><code>MuonPOG/Tools/test/crab/
</code></pre>

<p>The CRAB client can be sourced using the command below after cmsenv.</p>

<pre><code>source /cvmfs/cms.cern.ch/crab3/crab.sh
</code></pre>

<p>Check if you have writing permissions in the common area if you already asked for that. </p>

<pre><code>crab checkwrite --site=T2_CH_CERN --lfn=/store/group/phys_muon/
</code></pre>

<p>For running on the most recent re-reco data you can open file example: </p>

<pre><code>crab_SingleMuon_Run2016B-23SepReReco_Run273158.py    
</code></pre>

<p>whit the configuration pset parameters: </p>

<pre><code>config.JobType.pyCfgParams = ['globalTag=80X_dataRun2_2016SeptRepro_v3',
                              'ntupleName=muonPOGNtuple_SingleMuonRun2016B_23SepReReco.root',
                              'nEvents=-1',
                              'runOnMC=False',
                              'hltPathFilter=all',
                              'minMuPt=10.0',
                              'minNMu=2' ]
</code></pre>

<p>The ntuple producer gets loaded by :</p>

<div class="highlight highlight-source-python"><pre><span class="pl-k">from</span> MuonPOG.Tools.MuonPogNtuples_cff <span class="pl-k">import</span> appendMuonPogNtuple
appendMuonPogNtuple(process,<span class="pl-c1">False</span>,<span class="pl-s"><span class="pl-pds">"</span>HLT<span class="pl-pds">"</span></span>,<span class="pl-s"><span class="pl-pds">"</span>MY_NTUPLE_NAME.root<span class="pl-pds">"</span></span>)</pre></div>

<p>Where arguments are :</p>

<ol>
<li>The CMS configuration process</li>
<li>A bool to say whether you are running on MC</li>
<li>The label of the process giving HLT results</li>
<li>The name of the output ntuple</li>
</ol>
